## Key Invariants

1. All MCP tool responses use `blindedResponse()` with `[STUDENT_NNN]` tokens only. No real names in any content block.
2. Flat Zod schemas (`z.object()` only). `z.number()` for IDs (no `.int().positive()`).
3. Build order: core before teacher. New core exports added to `packages/core/src/index.ts`.
4. Tool registration follows `register*Tools(server, client, configManager, ...)` pattern.
5. Unit tests use MSW mocks, `McpServer` + `InMemoryTransport` + `Client.callTool()` pattern from `reporting.test.ts`.
6. Config deep-merge: new `attendance` section has defaults in `DEFAULT_CONFIG`.
7. Zoom name map keys are always lowercased.
8. Review file is overwritten on each parse (ephemeral).
9. Module-scoped `lastParseResult` holds parse state between parse and submit calls.
10. Submit clears `lastParseResult` after successful (non-dry-run) submission.

## Open Risks / Assumptions

1. **No vitest config for core package tests** — core tests may need to run via the teacher vitest config (which aliases `@canvas-mcp/core`), or a new vitest config for core may be needed. The Implementer should check if `packages/core/vitest.config.ts` exists; if not, either create one or run core unit tests from the teacher package.
2. **Zoom CSV format assumption** — The parser assumes Zoom CSV has columns `Name`, `User Email`, `Duration (minutes)`. If real Zoom CSVs vary, the parser may need adjustment. Test fixtures should be based on the format described in the brief.
3. **`CanvasClient.put` return type** — Assumed to return `Promise<T>` like `CanvasClient.post`. Needs verification if the Implementer encounters type issues.
4. **`packages/core/src/matching/` directory** — The brief references `name-index.ts` there but Glob found no files. The matching directory needs to be created fresh.
